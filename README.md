# 需求

P0:

- 对于每一个用户基于它自己的RSA公钥/私钥跟平台交互
- 用户首次注册个人账户
- 用户之间互相发送消息，平台作为中转加解密
- 平台将所有数据存储于一个公开透明的平台，比如GitHub、某云OSS
- CLI作为首个交互入口

P1:

- 平台安全性
  - username校验规则 防冒充
    - 用户名规则：支持英文字符、数字、.-_；具有一定规则避免冒充其他用户，比如peter，后面的人就必须加上一定的数字，比如peter421
- IM业务可靠性
  - msgId 防重复 幂等性
  - msgId ACK后再删除 防丢失
- 并发安全性
  - ratelimit 防暴力破解/DoS 基于ip等
- 性能/数据结构优化
  - 部分list改为map
- err http status
